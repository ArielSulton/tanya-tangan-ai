1. OVERVIEW & TUJUAN PIVOT
================================================================================

Platform Tanya Tangan AI dialihkan fungsinya dari platform aksesibilitas
layanan publik untuk tuna rungu di ruang publik, menjadi asisten belajar
cerdas yang dikhususkan untuk siswa Sekolah Luar Biasa tipe B (SLB-B /
Tunarungu).

Tagline baru:
  "Asisten belajar cerdas untuk siswa SLB-B berbasis bahasa isyarat SIBI"

Target pengguna:
  - Siswa SLB-B (end-user): bertanya materi pelajaran via gesture SIBI
  - Guru SLB-B (admin): upload konten kurikulum + monitor analitik siswa

YANG TIDAK BERUBAH (core logic tetap utuh):
  - SIBI gesture recognition (MediaPipe + TensorFlow.js)
  - RAG pipeline (LangChain + LLaMA 3.3 via ChatGroq + Pinecone)
  - FAQ clustering & recommendations (scikit-learn)
  - Admin dashboard architecture (Supabase Auth, role-based)
  - Monitoring stack (Prometheus + Grafana + DeepEval)
  - Speech-to-text sebagai input alternatif
  - QR code & ringkasan percakapan (PDF)
  - Session management, rate limiting, auth middleware

MAPPING KONSEP LAMA → BARU:
  institution        → subject (mata pelajaran)
  institution_name   → subject_name
  institution_slug   → subject_slug (format: {mp}-{jenjang}, contoh: matematika-smplb)
  Lembaga pemerintah → Mata pelajaran SLB-B
  Petugas pemerintah → Guru SLB-B
  Warga tuna rungu   → Siswa SLB-B
  Prosedur layanan   → Materi kurikulum + soal latihan


2. PENDEKATAN DESAIN YANG DIPILIH
================================================================================

Pendekatan B: Context-Enriched Pivot

Alasan pemilihan:
  - Menghormati constraint "tidak merubah base logic/flow"
  - Menghasilkan pivot yang meaningful untuk SLB, bukan sekadar rebrand
  - Perubahan teknis minimal: rename tabel + 2 field DB + rename metadata Pinecone
  - Nilai tambah nyata: jawaban AI lebih presisi karena filter jenjang + mapel
  - Analytics guru lebih bermakna (FAQ per mata pelajaran per jenjang)

Yang TIDAK dipilih:
  - Pendekatan A (terlalu minimal, melewatkan peluang SLB-spesifik)
  - Pendekatan C (terlalu banyak fitur baru, di luar scope pivot)


3. USER FLOWS
================================================================================

3.1 FLOW SISWA (END-USER)
--------------------------
Langkah 1 : Buka platform → Landing page baru (hero SLB-B)
Langkah 2 : Klik "Mulai Belajar" → Halaman pemilihan konteks (/layanan)
Langkah 3 : Pilih Jenjang: [SDLB] [SMPLB] [SMALB]         ← LANGKAH BARU
Langkah 4 : Pilih Mata Pelajaran: [Matematika] [B. Indonesia] [IPA] [IPS] [PKn] [dst.]
Langkah 5 : Masuk halaman komunikasi /komunikasi/[slug]
            Contoh slug: /komunikasi/matematika-smplb
Langkah 6 : Izinkan kamera → Lakukan gesture SIBI (atau Speech-to-Text)
Langkah 7 : Pertanyaan dikenali → dikirim ke FastAPI dengan konteks
            { mata_pelajaran: "Matematika", jenjang: "SMPLB" }
Langkah 8 : Pinecone query dengan filter metadata { subject_name, jenjang }
Langkah 9 : LLM generate jawaban kontekstual (bahasa sederhana, sesuai jenjang)
Langkah 10: Tampil jawaban + FAQ rekomendasi + opsi ringkasan (QR/PDF)

Catatan: Hanya langkah 3 (pilih jenjang) yang benar-benar baru.
         Semua langkah lain identik dengan flow lama, hanya konteks berubah.

3.2 FLOW GURU (ADMIN)
-----------------------
Langkah 1 : Login via Supabase Auth → Dashboard admin
Langkah 2 : Menu "Kelola Materi" → pilih mata pelajaran + jenjang
Langkah 3 : Upload dokumen (modul, silabus, soal latihan) → Pinecone embedding
Langkah 4 : Menu "Analitik Siswa" → lihat FAQ clustering per mata pelajaran
Langkah 5 : Menu "QA Log" → review pertanyaan siswa, evaluasi kualitas jawaban AI
Langkah 6 : Menu "Validasi Jawaban" → approve/reject respons AI yang meragukan
Langkah 7 : Menu "Manajemen Pengguna" → undang guru lain (invitation system tetap)


4. PERUBAHAN ARSITEKTUR & DATA MODEL
================================================================================

4.1 DATABASE (Drizzle ORM / PostgreSQL)
-----------------------------------------
Tabel: institutions → subjects

Field yang tetap:
  id, name, slug, description, created_at, updated_at

Field baru (ditambahkan):
  jenjang        ENUM('SDLB', 'SMPLB', 'SMALB') NOT NULL
  mata_pelajaran VARCHAR(100) NOT NULL

Field dihapus (tidak relevan untuk mapel):
  category, address, contact_info

4.2 PINECONE METADATA
-----------------------
Sebelum:
  { institution_id, institution_name, document_type }

Sesudah:
  {
    subject_id:      string,
    subject_name:    string,   // "Matematika", "Bahasa Indonesia", dll.
    jenjang:         string,   // "SDLB" | "SMPLB" | "SMALB"
    document_type:   string,   // "modul" | "silabus" | "soal_latihan" | "ringkasan_materi"
    grade:           string,   // "7" | "8" | "9" (opsional, untuk presisi)
    topic:           string,   // "Pecahan" | "Aljabar" (opsional)
    source:          string    // "Buku Teks Kemendikbud 2024"
  }

Query filter contoh:
  { subject_name: "Matematika", jenjang: "SMPLB" }

4.3 BACKEND API (FastAPI)
---------------------------
Endpoint rename:
  /api/v1/institutions → /api/v1/subjects

Parameter baru pada request RAG:
  Body tambah field "jenjang": "SDLB" | "SMPLB" | "SMALB"

File rename:
  institution_service.py → subject_service.py
  (logika sama, field disesuaikan)

4.4 FRONTEND API ROUTES (Next.js)
-----------------------------------
  /api/institutions       → /api/subjects
  /api/public/chat        → tambah parameter jenjang di request body

Slug format baru:
  /komunikasi/[slug] di mana slug = {mapel}-{jenjang}
  Contoh: /komunikasi/matematika-smplb
          /komunikasi/ipa-sdlb
          /komunikasi/bahasa-indonesia-smalb

4.5 KOMPONEN YANG TIDAK PERLU DIUBAH
---------------------------------------
  langchain_service.py         (hanya context string berubah)
  faq_clustering_service.py    (cluster by subject, logika sama)
  qa_logging_service.py        (struktur log tetap)
  pinecone_service.py          (hanya metadata key berubah)
  gesture recognition stack    (MediaPipe/TF.js tidak berubah)
  monitoring stack             (Prometheus/Grafana tidak berubah)
  auth middleware               (tidak berubah)
  session management           (tidak berubah)

SUMMARY SKALA PERUBAHAN:
  Layer                  | Perubahan                              | Skala
  -----------------------|----------------------------------------|--------
  DB schema              | Rename tabel + 2 field baru + hapus 3  | Kecil
  Pinecone metadata      | Ganti key names + tambah jenjang       | Kecil
  Backend endpoints      | Rename 1 endpoint + 1 parameter baru   | Kecil
  Frontend routes        | Rename + update slug format            | Kecil
  UI copy & komponen     | Rebrand semua label institusi → mapel  | Sedang
  Knowledge base         | Re-upload dokumen kurikulum SLB-B      | Di luar kode

6. KNOWLEDGE BASE & STRATEGI KONTEN RAG
================================================================================

6.1 STRUKTUR DOKUMEN YANG DIREKOMENDASIKAN
--------------------------------------------
Setiap dokumen di-embed ke Pinecone dengan metadata:
  {
    subject_name:  "Matematika",
    jenjang:       "SMPLB",
    document_type: "modul",
    grade:         "8",
    topic:         "Pecahan",
    source:        "Buku Teks Kemendikbud 2024"
  }

6.2 SUMBER KONTEN PER MATA PELAJARAN
---------------------------------------
  Mata Pelajaran      | Jenjang Target       | Sumber Direkomendasikan
  --------------------|----------------------|-----------------------------
  Matematika          | SDLB, SMPLB, SMALB  | Buku teks Kemendikbud adaptif SLB-B
  Bahasa Indonesia    | SDLB, SMPLB, SMALB  | Modul komunikasi & literasi SLB-B
  IPA                 | SMPLB, SMALB        | Silabus IPA adaptif SLB-B
  IPS                 | SMPLB, SMALB        | Modul IPS Kemendikbud SLB
  PKn                 | Semua jenjang        | Panduan kewarganegaraan adaptif

Catatan: Dokumen ini di-upload oleh guru melalui admin dashboard.
         Pipeline embedding ke Pinecone sudah tersedia — hanya metadata yang perlu disesuaikan.

6.3 SYSTEM PROMPT LLM (PERUBAHAN MINOR)
-----------------------------------------
Sebelum:
  "Kamu adalah asisten AI untuk membantu warga tunarungu mengakses
   layanan publik di {institution_name}. Jawab pertanyaan berdasarkan
   informasi resmi dari {institution_name}..."

Sesudah:
  "Kamu adalah asisten belajar untuk siswa tunarungu SLB-B.
   Jawab pertanyaan tentang {subject_name} untuk jenjang {jenjang}
   dengan bahasa yang sederhana, jelas, dan mudah dipahami siswa.
   Jika relevan, berikan contoh konkret atau langkah-langkah yang terurut.
   Gunakan konteks dokumen berikut untuk menjawab: {context}"

6.4 FAQ CLUSTERING (LOGIKA SAMA, DOMAIN BERUBAH)
--------------------------------------------------
Cluster yang diharapkan terbentuk secara otomatis:
  - "Operasi Bilangan"  → pertanyaan penjumlahan, pecahan, desimal
  - "Tata Bahasa"       → pertanyaan kalimat aktif/pasif, kata benda/kerja
  - "Siklus Hidup"      → pertanyaan metamorfosis, pertumbuhan tanaman
  - "Sejarah Nasional"  → pertanyaan kemerdekaan, tokoh pahlawan
  - "Nilai Pancasila"   → pertanyaan sila-sila dan pengamalannya

Algoritma scikit-learn clustering tidak berubah — domain FAQ yang berubah.

8. RISIKO & MITIGASI
================================================================================

  Risiko                              | Mitigasi
  ------------------------------------|------------------------------------------
  Dokumen kurikulum SLB-B belum ada   | Mulai dengan 2-3 mapel utama (Matematika,
                                      | B. Indonesia) sebelum expand
  Kualitas OCR buku teks rendah       | Prefer format digital (PDF resmi Kemendikbud)
  Slug collision mapel-jenjang        | Enforce unique constraint di DB
  Guru tidak familiar dengan platform | UI admin tetap sama → onboarding minimal
  SIBI model belum cover semua kosakata| Tetap sediakan speech-to-text & text input


9. KESIMPULAN
================================================================================

Pivot ini mengubah konteks platform dari layanan publik ke pendidikan SLB-B
tanpa menyentuh core logic yang sudah matang. Seluruh investasi teknis pada
gesture recognition, RAG pipeline, FAQ clustering, monitoring, dan admin
dashboard tetap dipertahankan dan langsung relevan di konteks baru.

Perubahan terbesar justru ada di lapisan konten (dokumen kurikulum) dan
UI copy — bukan di kode inti. Ini menjadikan pivot ini efisien secara
engineering sekaligus meaningful secara dampak sosial bagi ekosistem SLB-B
di Indonesia.
