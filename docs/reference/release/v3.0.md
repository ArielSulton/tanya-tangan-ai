# PENSyarat AI v3.0 — Visual Vocabulary Platform for SDLB-B

> **Release**: 20 April 2026
> **Codename**: TanyaTangan Visual  
> **Stack**: Next.js 15 (App Router) + FastAPI + PostgreSQL (Supabase) + Drizzle ORM

---

## Apa yang Berubah dari v2.x

v2.x adalah platform chatbot RAG per mata pelajaran (jenjang × mapel) untuk siswa tunarungu. v3.0 melakukan **pivot total** menjadi platform kosakata visual dengan pengenalan gestur SIBI.

### Perubahan Utama

| Area | v2.x (Sebelumnya) | v3.0 (Sekarang) |
|---|---|---|
| Fokus utama | Chatbot RAG per mapel | Kosakata visual + gestur SIBI |
| Input pengguna | Teks/voice chat | **Gestur tangan SIBI** (MediaPipe + TF.js) + teks manual |
| RAG | Pinecone per subject namespace | **Tidak ada RAG** — murni kosakata dari DB |
| Halaman utama | `/komunikasi/[slug]` (chat) | `/vocab` (kategori) → `/vocab/[kategori]` (gestur + hasil) |
| Interaktivitas | Chat bubbles | **Komponen interaktif per tipe kata** (slider, timeline, dial, gauge, drag-drop quiz) |
| Dashboard admin | Hanya QA logs | QA logs + **Manajemen kosakata** + **Permintaan kosakata** (unmatched inputs) |
| Correction AI | Tidak ada | **3-layer typo correction** (SIBI confusion → fuzzy → LLM) |

  ├─ GET  /api/v1/vocab/categories

## Fitur Baru v3.0

### 1. Pengenalan Gestur SIBI Real-time

- **MediaPipe Hands** + **TensorFlow.js** + **FingerPose** berjalan langsung di browser
- File model: `frontend/public/models/` (`.h5` + `.bin`)
- Komponen: `frontend/src/components/gesture/gesture-recognition.tsx`
- Deteksi huruf SIBI → formasi kata → lookup ke DB
- Input manual via keyboard juga tersedia

### 2. Penanganan Kontekstual Kata Keterangan (Adverb)

Setiap sub-tipe keterangan mendapat komponen interaktif berbeda:

| Sub-tipe | Komponen | Interaksi |
|---|---|---|
| `degree` | `<IntensitySlider>` | Slider kontinu → elemen visual berubah ukuran + emoji scaling |
| `temporal` | `<TimelineAnimation>` | Timeline interaktif dengan ikon berulang sesuai frekuensi |
| `modality` | `<CertaintyDial>` | Conic gradient donut chart dengan emoji + persentase |
| `intensity` | `<SensationGauge>` | Progress bar gauge dengan emoji transitions + sensation badge |

**Dispatch logic** (`frontend/src/lib/adverb-types.ts`):
- Kata dengan `adverb_subcategory` + config → komponen interaktif
- Kata abstrak dengan `comparison` saja → `<AbstractComparison>` (fallback statis)
- Kata tanpa keduanya → empty state

### 3. Mode Kuis

#### `/vocab/kata_keterangan` → DragDropQuiz
- Emoji-based drag & drop dari tiang jawaban ke area jawaban
- Keterangan cocokkan dengan lawan katanya (sangat ↔ sedikit)
- Feedback visual: confetti emoji rain untuk jawaban benar, 🟥 untuk salah
- **Tanpa audio** — semua feedback visual/haptic untuk siswa SDLB-B

#### `/vocab/hewan|benda|alam|perasaan` → ImageMatchQuiz
- Drag & drop gambar ke drop zone sesuai teks yang ditampilkan
- 4 opsi gambar per soal (1 benar + 3 distraktor)
- Hanya kata `konkret` dengan `image_url` yang muncul
- Menggunakan `@dnd-kit/core` (sama library dengan DragDropQuiz)
- Fallback emoji jika gambar gagal dimuat (React state-based, bukan innerHTML)

### 4. Koreksi Typo 3-Layer

Saat gestur/input tidak cocok langsung di DB, cascade berjalan:

1. **Layer 1 — SIBI Confusion Map** (instan, zero-latency)
   - `backend/app/services/typo_correction_service.py`
   - Per-letter confusion mapping (A↔H, B↔R, D↔T, dll)
   - Full-word corrections (kaop→kucing, sgat→sangat, dll)
   - Variant generation dari letter-swap

2. **Layer 2 — DB Fuzzy Lookup** (Jaccard 0.7 + Levenshtein 0.3)
   - `_fuzzy_lookup_enhanced()` di `vocab_service.py`
   - Mencari kata terdekat dalam kategori yang sama
   - Kata pendek menggunakan weight 0.4/0.6

3. **Layer 3 — LLM Correction** (ChatGroq LLaMA 3.3 70B)
   - `_llm_correct()` di `vocab_service.py`
   - Konteks: daftar kata dalam kategori
   - Output format: "KATA|PENJELASAN"
   - Child-friendly explanation

**Cascade**: Layer 1 → Layer 2 → Layer 1+fuzzy combo → Layer 3 → fallback

### 5. Permintaan Kosakata (Admin Dashboard)

- Setiap input gestur yang tidak cocok dicatat ke tabel `word_requests`
- Admin bisa melihat daftar input yang belum ditangani (`status=pending`)
- Tombol "Tambah Kata" di setiap baris → membuka dialog tambah kosakata yang sudah ada dengan `wordText` terisi
- Filter: Semua / Belum Ditangani / Sudah Ditangani
- Endpoint: `GET /api/v1/vocab/requests` (backend) + `GET /api/admin/vocab/requests` (frontend Next.js route)

## Alur Pengguna (User Flow)

### Siswa (Public, Tanpa Login)

```
Landing Page → /vocab (kategori grid)
  → Pilih kategori → /vocab/[kategori]
    → Mode Jelajah: Nyalakan kamera → gestur SIBI → hasil kata
      → konkret → ConcreteWordCard (gambar + label)
      → abstrak + adverb → komponen interaktif sesuai sub-tipe
      → abstrak tanpa config → AbstractComparison atau empty state
      → tidak ditemukan → AI Fallback (3-layer correction)
    → Mode Kuis (toggle button):
      → kata_keterangan → DragDropQuiz (emoji drag & drop)
      → lainnya → ImageMatchQuiz (gambar drag & drop)
```

### Admin (Login JWT)

```
Dashboard → Manajemen Kosakata
  → Tambah/Edit/Hapus kata (konkret + gambar, abstrak + comparison, keterangan + config)
  → Lihat daftar kata per kategori

Dashboard → Logs & System Monitoring
  → QA Logs (chatbot lama)
  → Permintaan Kosakata (gesture input yang tidak cocok)
    → Filter: Semua / Belum Ditangani / Sudah Ditangani
    → Tombol "Tambah Kata" per baris
```

## Konfigurasi Penting

| Variabel | Nilai | Catatan |
|---|---|---|
| `ENVIRONMENT` | `development` / `production` | Production menonaktifkan Swagger dan mengubah CORS |
| `BACKEND_URL` | `http://localhost:8000` (dev) / `http://backend:8000` (docker) | Proxy Next.js → FastAPI |
| `GROQ_API_KEY` | diperlukan | Untuk Layer 3 LLM correction |
| `DATABASE_URL` | Supabase PostgreSQL | Drizzle ORM (frontend) + SQLAlchemy (backend) |
| `PINECONE_API_KEY` | tidak digunakan | v3.0 tidak menggunakan RAG |

### Perhatian

- Frontend `next.config.ts` memiliki `typescript.ignoreBuildErrors: true` — **selalu jalankan `bun run tsc` manual** sebelum mengklaim frontend selesai
- Backend `config.py` membangun SQLAlchemy URL dari variabel individual (user, password, host, port, dbname), bukan `DATABASE_URL` langsung
- `SECRET_KEY` backend harus ≥32 karakter
- `.gitignore` mengecualikan file model `.h5` dan `.bin` kecuali `model.h5`

## Perintah Penting

```bash
# Start semua service (development)
COMPOSE_BAKE=true docker compose -f compose.dev.yaml up --build

# Frontend only
cd frontend && bun run dev       # Turbopack dev server (port 3000)
cd frontend && bun run tsc        # Type-check WAJIB sebelum klaim selesai
cd frontend && bun run lint      # ESLint

# Backend only
cd backend && source .venv/bin/activate
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# Database migration
cd frontend && bun run db:generate   # Generate migration
cd frontend && bun run db:migrate    # Apply migration
cd frontend && bun run db:studio     # Drizzle Studio GUI

# Seed vocab data
cd backend && python scripts/seed_vocab.py
```

## Constraint Desain

- **Tanpa audio** — semua feedback untuk siswa SDLB-B harus visual/haptic (confetti, warna, emoji animation)
- **Tangan saja** — tidak ada NMM Face Mesh detection, hanya gestur tangan SIBI
- **Konteks-dependent treatment** — setiap sub-tipe keterangan punya komponen interaktif berbeda
- **Quiz** — drag & drop, bukan multiple choice (kinesthetic learning untuk tunarungu)
- **Indonesian language** — platform khusus peningkatan bahasa Indonesia, tidak per-mapel
