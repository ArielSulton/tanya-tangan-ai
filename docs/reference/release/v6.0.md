> **Release**: 17 Mei 2026

> **Stack**: Next.js 15 (App Router) + React 19 + Bun + FastAPI + PostgreSQL (Supabase) + Drizzle ORM + @tensorflow-models/hand-pose-detection (MediaPipeHands runtime) + Fingerpose + TFJS classifier (MLP / LSTM)

## Apa yang Berubah dari v5.0

v5.0 memindahkan gesture recognition ke server-side (YOLO + MediaPipe Python). v6.0 **memigrasi kembali ke client-side** dengan stack browser-native (MediaPipe Hands + TFJS), menambahkan dukungan dynamic gestures (word-level), dan membangun infrastruktur data-collection terdistribusi yang bisa di-share ke kontributor.

### Perubahan Utama

| Area | v5.0 (Sebelumnya) | v6.0 (Sekarang) |
|---|---|---|
| Gesture Recognition | YOLO + MediaPipe Python (backend) | MediaPipe Hands + Fingerpose/MLP/LSTM (browser) |
| Processing Location | Server-side (FastAPI) | Client-side (TFJS in browser) |
| Network Round-trip | Per frame (HTTP POST) | Zero вЂ” inference fully offline-capable |
| Static Path | YOLO classification | Fingerpose rule-based (default) atau TFJS MLP (env-toggled) |
| Dynamic Path | N/A | MotionDetector + 32-frame buffer + TFJS LSTM (Kazuhito-style) |
| Validation | Temporal frame buffer di backend | Dwell-time (~0.8s hold) static + motion_end trigger dynamic |
| Data Collection | Manual via record di /dev only | Distributed: standalone HTML + bulk image-import dari folder |
| Training Pipeline | Out-of-repo custom YOLO | In-repo: `train_static.ipynb` (MLP) + `train_dynamic.ipynb` (LSTM) + TFLiteв†’TFJS conversion |
| Legacy v5.0 (YOLO + backend gesture) | Active runtime | Preserved on disk, route tidak di-wire |

## Fitur Baru v6.0

### 1. Browser-Native Gesture Pipeline

Gesture recognition kembali sepenuhnya ke browser. Stack:

- **`@tensorflow-models/hand-pose-detection` v2.0.1** вЂ” wrapper resmi MediaPipe Hands untuk JS (runtime: `mediapipe` + asset dari JSDelivr CDN)
- **Fingerpose** вЂ” rule-based gesture estimator (default static engine)
- **TFJS** (`@tensorflow/tfjs` + `@tensorflow/tfjs-backend-webgl`) вЂ” runtime untuk MLP/LSTM classifier yang ditrain dari notebook in-repo

**Service layer flow:**
```ts
// frontend/src/lib/gesture/engine.ts (BrowserGestureEngine)
handleRawHands(raws) {
  // Static path: per-frame
  const pair = sortHandsByXPosition(raws)
  const features = extractFrameFeatures(pair)  // 84 floats
  if (mlpReady) runStaticInference(features)    // MLP path
  // Else: fingerpose runs via GestureRecognitionService.setOnResult

  // Dynamic path: per-frame buffer push
  const pxPoint = { x: rawSlot0.x, y: rawSlot0.y }
  this.motionDetector.update(pxPoint)            // pixel-space variance
  this.dynamicHistory.push({ x: pxPoint.x / vw, y: pxPoint.y / vh })  // [0,1] for model

  if (motionDetector.state === 'motion_end' && history.length === 32) {
    runDynamicInference(history)                 // emit word
  }
}
```

### 2. Static Engine Selector

Engine static bisa di-switch antara **fingerpose (rule-based)** dan **MLP (TFJS)** via env var `NEXT_PUBLIC_STATIC_ENGINE`:

| Mode | Default | Behavior |
|---|---|---|
| `fingerpose` | вњ… default | Rule-based dari `/components/handsigns/` (Asign.ts, Bsign.ts, ..., Ysign.ts) |
| `mlp` | opt-in | Load `/models/static/model.json` в†’ 84в†’Dense(40)в†’Dense(20)в†’Dense(24) softmax. Auto-fallback ke fingerpose kalau model file 404 |

Konfigurasi build-time (baked saat `next build`):
```bash
NEXT_PUBLIC_STATIC_ENGINE=mlp docker compose -f compose.local.prod.yaml up --build
```

Env var di-wire lewat:
- `.env.example`
- `frontend/Dockerfile.prod` (ARG + ENV + secret mount)
- `compose.dev.yaml` / `compose.prod.yaml` / `compose.local.prod.yaml`

### 3. Dynamic Gesture Path

Word-level gesture recognition baru via:

**MotionDetector** (`frontend/src/lib/gesture/motion-detector.ts`):
- Hysteresis state machine: `idle в†’ still в†’ moving в†’ motion_end в†’ still`
- 15-frame rolling window, variance threshold 2000 (start) / 400 (end) di pixel space
- Fire `motion_end` event sekali per gerakan selesai

**Dynamic Classifier** (`frontend/src/lib/gesture/inference/dynamic-classifier.ts`):
- Load TFJS graph model dari `/models/dynamic/model.json`
- Input: 64-float (32 frames Г— 2 coords) вЂ” wrist trajectory di [0,1] space
- Output: softmax atas 7 kelas (`jeruk, kucing, besar, kecil, sangat, j, z`)
- Confidence threshold default 0.6
- Lazy-load + cache-failure (404 в†’ silent fallback, no retry spam)

**Kazuhito-style LSTM** (`training/train_dynamic.ipynb`):
```python
model = Sequential([
    InputLayer(input_shape=(FEATURE_LENGTH,)),  # 64
    Reshape((TIME_STEPS, DIMENSION)),            # (32, 2)
    Dropout(0.2),
    LSTM(16, input_shape=[TIME_STEPS, DIMENSION]),
    Dropout(0.5),
    Dense(10, activation='relu'),
    Dense(NUM_CLASSES, activation='softmax'),
])
```

### 4. Auto-Disambiguate Static vs Dynamic

Dua path jalan paralel tiap frame tanpa user toggle. Dua filter alami yang memisahkan:

| Kondisi User | Static Path | Dynamic Path | Output |
|---|---|---|---|
| Hold "A" 1 detik | вњ“ "A" terus в†’ dwell-validate | вњ— no motion_end | **A** |
| Gerakan "jeruk" в†’ diam | вњ“ tapi pose berubah в†’ ga validate | вњ“ motion_end fire в†’ "jeruk" | **jeruk** |
| Hold rapid: K, U, C, I, N, G | вњ“ tiap huruf hold 0.8s | вњ— | **KUCING** |
| Diam tapi acak (no clean pose) | вњ“ flicker в†’ ga validate | вњ— | (nothing) |

`BrowserGestureResult.gestureType: 'static' | 'dynamic'` discriminator agar hook tau cara validate:
- **Static**: dwell-validation 24 consecutive frames at в‰Ґ0.7 confidence (~0.8s @ 30fps)
- **Dynamic**: auto-validate first emission yang clear confidence threshold (single fire per motion_end)

### 8. Training Pipeline (In-Repo)

Notebook + script lengkap di `training/`:

```
training/
в”њв”Ђв”Ђ README.md
в”њв”Ђв”Ђ train_static.ipynb              в†’ 84в†’Dense(40)в†’Dense(20)в†’Dense(24) MLP (Kazuhito-style)
в”њв”Ђв”Ђ train_dynamic.ipynb             в†’ Reshapeв†’LSTM(16)в†’Dense(10)в†’Dense(7) (Kazuhito's LSTM variant)
в”њв”Ђв”Ђ keypoint_csv/
в”‚   в”њв”Ђв”Ђ sibi_label.csv              в†’ 24 labels A-Y
в”‚   в””в”Ђв”Ђ sibi.csv                    в†’ static training data (gitignored)
в”њв”Ђв”Ђ point_history_csv/
в”‚   в”њв”Ђв”Ђ dynamic_label.csv           в†’ 7 labels (jeruk/kucing/besar/kecil/sangat/j/z)
в”‚   в””в”Ђв”Ђ dynamic.csv                 в†’ dynamic training data (gitignored)
в”њв”Ђв”Ђ convert_to_tfjs.sh              в†’ TFLite в†’ TFJS graph model + labels.json
в””в”Ђв”Ђ merge_csvs.py                   в†’ Aggregate contributor CSVs into single dataset
```

**CSV format**:
- Static: `label,f0,f1,...,f83` (85 kolom)
- Dynamic: `label,x0,y0,x1,y1,...,x31,y31` (65 kolom)

**Workflow**:
1. Collect samples (via `/dev/` atau standalone, atau bulk-import dari Kaggle)
2. Export CSV
3. Drop ke `training/keypoint_csv/sibi.csv` / `training/point_history_csv/dynamic.csv`
4. Run `merge_csvs.py contributions/` kalau collect dari multiple kontributor
5. Run notebook в†’ produces `.hdf5` + `.tflite`
6. Run `convert_to_tfjs.sh` в†’ produces `model.json` + `group1-shard*.bin` + `labels.json` di `frontend/public/models/{static,dynamic}/`
7. Reload browser в†’ model auto-load (dynamic always wired, static gated by `NEXT_PUBLIC_STATIC_ENGINE=mlp`)

### 9. CSV Merge Tool

`training/merge_csvs.py` вЂ” aggregate contributor CSVs:

```bash
python3 training/merge_csvs.py contributions/
```

- Auto-routes CSVs by filename pattern (`sibi_static_*.csv` vs `sibi_dynamic_*.csv`) AND column count (85 vs 65) as fallback
- Skips malformed/non-recorder files with warning
- Per-class sample distribution report вЂ” spot class imbalance early
- Default output: `training/keypoint_csv/sibi.csv` + `training/point_history_csv/dynamic.csv`

### 10. MediaPipe Bundler Shim

`@mediapipe/hands` npm package ships sebagai UMD script yang attach `Hands` ke `window` via side-effect вЂ” strict prod bundler (webpack/turbopack) gagal pada `import { Hands } from "@mediapipe/hands"` yang dilakukan oleh `@tensorflow-models/hand-pose-detection`.

**Fix**:
- `frontend/src/lib/gesture/mediapipe-hands-shim.ts` вЂ” class wrapper yang delegate ke `window.Hands` at construction time + async `loadHandsScript()` yang inject CDN UMD script
- `frontend/next.config.ts` вЂ” webpack `resolve.alias` + turbopack `resolveAlias` map `@mediapipe/hands` в†’ shim
- `HandPoseService.initialize()` вЂ” `await loadHandsScript()` before `createDetector()`

Tanpa shim, `next build` crash dengan `"Export 'Hands' doesn't exist in target module"`.

### 11. Hand Dedupe + Skeleton Overlay

**Dedupe**: MediaPipe kadang emit 2 deteksi untuk 1 tangan fisik (palm vs landmark stage disagree). `dedupeOverlappingHands()` filter di mana wrist within 80px (or [0,1] equivalent) treated as duplicate, keep higher confidence.

**Skeleton overlay**: `/dev/gesture-recorder` + standalone gambar 21 landmarks + 5 finger chains di canvas overlay. Red dots untuk thumb (0-3), blue untuk jari lain (4-20), green lines untuk connections.

### 12. Layout Match Vocab Style

`/dev/gesture-recorder` di-restyle agar match `/vocab/[kategori]` visual:
- Camera: double-rounded wrap (2.5rem outer + 2rem inner) + shadow-inner + ring
- Container: `max-w-7xl mx-auto` + `grid-cols-12 items-stretch`
- Sample list: `flex-1` stretch в†’ bottom-aligned dengan kolom kiri
- Header centered ke max-w-7xl

### 13. Coord-Space Normalization Fix

Pre-v6.0 inconsistency: production engine pushed PIXEL wrist coords, standalone pushed NORMALIZED [0,1] в†’ trained model only worked for one source.

**Fix**:
- `engine.ts`: MotionDetector tetap dapat pixel (sensitivity threshold), dynamic history sekarang [0,1] (divide by `video.videoWidth/videoHeight`)
- `/dev/gesture-recorder`: same divide before push ke recorder
- Standalone: already [0,1] sejak awal

`BrowserGestureResult.gestureType` discriminator + hook auto-validate dynamic results agar dynamic word actually masuk ke "kata terbentuk" di `/vocab/*`.

### 14. Dwell-Time Validation + Stale Result Clear

`use-gesture-recognition.ts`:
- Static: letter hanya append ke kata terbentuk kalau **same letter persisted 24 frames** (`VALIDATION_FRAMES`) di confidence в‰Ґ 0.7. Prevent flicker.
- Stale clear: kalau engine ga emit hasil baru dalam 600ms в†’ `lastResult = null`. Badge UI hilang pas tangan ilang dari kamera.
- Dynamic: auto-validate single emission yang clear confidence floor (bypass streak).

| Variabel | Nilai | Catatan |
|---|---|---|

- **Tanpa audio** вЂ” feedback siswa SDLB-B harus visual (confetti, warna, badge huruf)
