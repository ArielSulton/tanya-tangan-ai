# PENSyarat AI v5.0 — YOLO + MediaPipe Backend Gesture Recognition

> **Release**: 23 April 2026
> **Codename**: TanyaTangan Neural
> **Stack**: Next.js 15 (App Router) + FastAPI + PostgreSQL (Supabase) + Drizzle ORM + MediaPipe Python + Ultralytics YOLOv8

---

## Apa yang Berubah dari v4.0

v4.0 memperkenalkan visual kata keterangan dan quiz rework. v5.0 memigrasikan **gesture recognition dari client-side (TFJS + MediaPipe JS) ke server-side (YOLO + MediaPipe Python)**.

### Perubahan Utama

| Area | v4.0 (Sebelumnya) | v5.0 (Sekarang) |
|---|---|---|
| Gesture Recognition | TFJS + MediaPipe JS (browser) | YOLOv8 + MediaPipe Python (backend) |
| Processing Location | Client-side (browser) | Server-side (FastAPI) |
| Frontend Role | Full processing + display | Camera capture → send frame → display result |
| Model | TFJS model (.h5) + FingerPose | YOLOv8 custom model (.pt) |
| YOLO Training | N/A | Custom trained on SIBI hand signs |
| Legacy Code | N/A | TFJS + MediaPipe JS code **kept on disk** (not running) |

## Fitur Baru v5.0

### 1. YOLOv8 + MediaPipe Python Backend

Gesture recognition now runs server-side using:
- **MediaPipe Python** (`mediapipe>=0.10.0`) — hand landmark extraction
- **Ultralytics YOLOv8** (`ultralytics>=8.0.0`) — gesture classification

**New backend services:**
```
backend/app/services/
├── mediapipe_service.py         → MediaPipe Hands wrapper (hand landmark extraction)
├── yolo_service.py              → YOLOv8 model wrapper (gesture classification)
└── gesture_recognition_service.py  → Orchestrates MediaPipe + YOLO pipeline
```

**Service layer flow:**
```python
# gesture_recognition_service.py
def recognize(image: np.ndarray) -> Dict:
    # Step 1: MediaPipe - extract hand landmarks
    landmarks = self._mediapipe.extract_landmarks(image)
    if landmarks is None:
        raise ValueError("No hand detected in the image")

    # Step 2: YOLO - classify gesture
    prediction = self._yolo.predict_letter(image, conf_threshold=0.5)
    if prediction is None:
        raise ValueError("No gesture recognized")

    return {
        "letter": prediction["letter"],
        "confidence": prediction["confidence"],
        "alternatives": alternatives[:2],
        "processing_time_ms": processing_time_ms,
    }
```

### 2. Custom YOLO Model for SIBI

- **Model**: `sibi_yolo.pt` — custom YOLOv8 trained on SIBI hand signs
- **Classes**: 25 letters (A-Z, J→Y per SIBI mapping)
- **Location**: `backend/app/models/sibi_yolo.pt`
- **Format**: PyTorch `.pt` (Ultralytics format)
- **Gitignored**: Yes (too large for git)

**Class to letter mapping:**

### 3. Frontend Thin Client

Frontend no longer processes gestures locally. Instead:
1. **Camera capture** via `navigator.mediaDevices.getUserMedia`
2. **Frame to base64** via `canvas.toDataURL('image/jpeg', 0.8)`
3. **POST to backend** at `/api/v1/gesture/recognize`
4. **Display result** from backend response

**Modified files:**
- `frontend/src/hooks/use-gesture-recognition.ts` — sends frames to backend instead of local TFJS
- `frontend/src/components/gesture/gesture-recognition.tsx` — uses backend API response

```json
// 422: No hand detected
```

## Konfigurasi Penting

| Variabel | Nilai | Catatan |
|---|---|---|
| `ENVIRONMENT` | `development` / `production` | Production menonaktifkan Swagger dan mengubah CORS |
| `BACKEND_URL` | `http://localhost:8000` (dev) / `http://backend:8000` (docker) | Proxy Next.js → FastAPI |
| `YOLO_MODEL_PATH` | `/app/models/sibi_yolo.pt` | Path ke YOLO model di container |

### Perhatian

- Frontend `next.config.ts` memiliki `typescript.ignoreBuildErrors: true` — **selalu jalankan `bun run tsc` manual** sebelum klaim frontend selesai
- Backend `config.py` membangun SQLAlchemy URL dari variabel individual (user, password, host, port, dbname), bukan `DATABASE_URL` langsung
- `SECRET_KEY` backend harus ≥32 karakter
- `.gitignore` mengecualikan file model `.h5`, `.bin`, dan `sibi_yolo.pt`
- YOLO model harus di-train sendiri dengan dataset SIBI hand signs
